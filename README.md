## Постановка задачи

Необходимо разработать модуль лексического анализа, который выполняет разбор исходного текста программы.

Лексический анализатор должен:

- принимать на вход текст программы;
- выделять лексемы;
- классифицировать их по типам;
- определять местоположение каждой лексемы (номер строки и позиции символов);
- обнаруживать недопустимые символы и выводить сообщение об ошибке;
- поддерживать многострочный ввод.

Результаты анализа выводятся в таблицу со следующими столбцами:

- условный код лексемы;
- тип лексемы;
- лексема;
- местоположение.

## Краткое описание работы автомата

Работа автомата начинается из **начального состояния**.

В зависимости от типа входного символа происходит переход в соответствующее состояние:

- буква → состояние чтения идентификатора;
- цифра → состояние чтения числа;
- кавычка → состояние строкового литерала;
- пробел → состояние пробельных символов;
- специальные символы (`<`, `>`, `{`, `}`, `,`, `;`, `=`) → отдельные состояния.

После завершения распознавания лексемы автомат возвращается в начальное состояние и продолжает анализ текста.

Если встречается символ, не принадлежащий ни одному допустимому типу, фиксируется **лексическая ошибка**.
